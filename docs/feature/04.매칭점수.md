# 매칭 점수 기준 (4단계)

## 총점 공식

```
총점 = 거리점수 × 100 + 품목점수 × 10 + 크기점수 × 1 + 가격점수 × 5
```

| 구성 요소 | 가중치 | 범위 | 최대 기여 |
|-----------|--------|------|-----------|
| 거리점수   | ×100   | 0–100 | 10,000점 |
| 품목점수   | ×10    | 0–100 | 1,000점  |
| 크기점수   | ×1     | 0–100 | 100점    |
| 가격점수   | ×5     | 0–100 | 500점    |
| **합계**  |        |       | **11,600점** |

---

## 항목별 계산 방식

### 거리점수 (dist_score)
```
거리점수 = max(0, 100 - distance_km)
```
- 1km: 99점 · 50km: 50점 · 100km: 0점 (100km 이상은 0점)
- PostGIS `ST_DWithin`으로 max_km 초과 시 애초에 제외

### 품목점수 (species_score)
```
품목점수 = max(0, 100 - 개념 공간 유클리드 거리)
```
- `species_space` 테이블에 2D 좌표로 수종 분류 저장
  - 침엽수 (0–80): 소나무(0,0), 잣나무(10,5), 측백나무(20,0) ...
  - 활엽수 (200–250): 느티나무(200,0), 왕벚나무(210,5) ...
  - 관목 (300–340): 철쭉(300,0), 영산홍(305,0) ...
- **같은 수종**: 100점 · **같은 분류 내**: 50–90점 · **다른 분류**: 10점 이하
- 품목점수 < 10점이면 매칭 후보에서 제외

### 크기점수 (size_score)
우선순위 순서로 첫 번째 유효한 값으로 계산:
1. **근원직경 (★ 최우선)**: `max(0, 100 - |공급R - 수요R| × 5)` · 5cm 차이 = -25점
2. **흉고직경**: `max(0, 100 - |공급B - 수요B| × 5)` · 동일 방식
3. **수령**: `max(0, 100 - |공급수령 - 수요수령| × 80)` · 1년 차이 = -80점
4. **수고+수관폭 2D**: `max(0, 100 - √((H차이)² + (W차이)²) × 20)`
5. 모두 없으면: 50점 (중립)

### 가격점수 (price_score)
```
예산단가 = 수요.예산 / 수요.수량
가격점수 = max(0, min(100, (1 - 공급단가 / 예산단가) × 100))
```
- 단가 = 예산단가 → 0점 (예산 한도에 딱 맞음)
- 단가 = 예산단가의 50% → 50점
- 단가 → 0 → 100점 (최고 절감)
- 단가 > 예산단가인 경우: 하드 필터에서 애초에 제외

---

## 하드 필터 (매칭 후보에서 완전 제외)

| 조건 | 설명 |
|------|------|
| `s.quantity < d.quantity` | 공급 수량이 수요 수량보다 부족 |
| `s.unit_price × d.quantity > d.budget` | 총 금액이 예산 초과 |
| 거리 > max_km | 설정 거리 초과 |
| 품목점수 < 10 | 수종이 완전히 다른 분류 |

## 정렬 기준 선택

UI에서 4가지 정렬 기준 선택 가능:

| 기준 | 설명 | 방향 |
|------|------|------|
| 총점순 (기본) | 종합 점수 높은 순 | 내림차순 |
| 거리순 | 가까운 순 | 오름차순 |
| 가격순 | 가격 절감률 높은 순 | 내림차순 |
| 수종순 | 수종 유사도 높은 순 | 내림차순 |

## 관리자 수동 추천 매칭

- 관리자는 `/admin/matches`에서 수동으로 demand + stock 쌍을 매칭할 수 있다
- `is_manual = true`, `reason` 필드에 추천 이유를 저장
- 매칭 카드에서 "추천" 뱃지와 이유가 표시됨
- API: `POST /api/matches/admin/manual` (관리자 전용)

## 향후 개선 예정

- **공급자 신뢰도 점수**: 거래 완료 건수 기반. 초기 10% 반영 후 단계적 확대 → [거래 신뢰도 시스템](05.신뢰도시스템.md) 참고
- **수량 초과 보너스**: 공급 수량이 수요 수량의 2배 이상일 때 소폭 가산
- **긴급 수요 가중치**: is_urgent = true일 때 거리 가중치 추가 적용
