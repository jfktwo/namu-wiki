# 거래 흐름 및 평가 화면

## 전체 흐름 요약

```
문의 생성 → 판매자 수락 → 견적서 작성 → 구매자 수락
  → 납품 완료 → 정산 완료 → 평가
```

---

## 단계별 상세

### 1단계 · 문의 생성 (구매자)

- 매칭 카드에서 "문의하기" 버튼 클릭
- `inquiries.status = 'requested'`

### 2단계 · 문의 수락 (판매자)

- 문의 상세 페이지에서 **수락하기** 클릭
- `inquiries.status = 'accepted'`
- 이 시점부터 견적서 섹션이 노출됨

### 3단계 · 견적서 작성 (판매자)

- 수락 후 문의 상세 페이지 하단 견적서 섹션에서 작성
- 수량, 단가, 운송비, 납품 예정일, 납품 조건, 메모 입력
- `transactions.status = 'pending'`

### 4단계 · 견적서 수락 (구매자)

- 구매자가 견적 내용 확인 후 **수락하기** 클릭
- `transactions.status = 'confirmed'`

### 5단계 · 납품 완료 (판매자)

- 실제 납품 후 판매자가 **납품 완료** 버튼 클릭
- `transactions.status = 'delivered'`

### 6단계 · 정산 완료 (구매자)

- 납품 확인 후 구매자가 **정산 완료** 버튼 클릭
- `transactions.status = 'settled'`
- **이 시점부터 평가 섹션이 자동으로 노출됨**

### 7단계 · 상호 평가

- 구매자 → 판매자 평가 (별점 + 태그)
- 판매자 → 구매자 평가 (별점 + 태그)
- 평가 완료 시 `reviews` 테이블에 저장, `users.trust_score` 자동 갱신

---

## 상태 전이 다이어그램

```
inquiries.status
  requested ──[판매자 수락]──▶ accepted ──[판매자 거절]──▶ rejected
      └──[구매자 취소]──▶ cancelled

transactions.status (inquiries.status = 'accepted' 이후)
  pending ──[구매자 수락]──▶ confirmed ──[판매자]──▶ delivered ──[구매자]──▶ settled
      └──[구매자 거절]──▶ cancelled
```

---

## 권한 정리

| 액션 | 실행 가능한 역할 | 조건 |
|------|----------------|------|
| 문의 수락/거절 | 판매자 (receiver) | `status = 'requested'` |
| 문의 취소 | 구매자 (requester) | `status = 'requested'` |
| 견적서 작성 | 판매자 | `inquiry.status = 'accepted'`, 견적 없음 |
| 견적서 수락 | 구매자 | `tx.status = 'pending'` |
| 견적서 거절 | 구매자 | `tx.status = 'pending'` |
| 납품 완료 | 판매자 | `tx.status = 'confirmed'` |
| 정산 완료 | 구매자 | `tx.status = 'delivered'` |
| 평가 제출 | 구매자 / 판매자 | `tx.status = 'settled'`, 거래당 1회 |

## API 엔드포인트

| 메서드 | 경로 | 설명 |
|--------|------|------|
| `POST` | `/api/inquiries/{id}/accept` | 문의 수락 |
| `POST` | `/api/inquiries/{id}/reject` | 문의 거절 |
| `POST` | `/api/transactions` | 견적서 생성 |
| `POST` | `/api/transactions/{id}/confirm` | 견적서 수락 |
| `POST` | `/api/transactions/{id}/cancel` | 견적서 거절 |
| `POST` | `/api/transactions/{id}/deliver` | 납품 완료 처리 |
| `POST` | `/api/transactions/{id}/settle` | 정산 완료 처리 |
| `POST` | `/api/reviews` | 평가 제출 |
| `GET` | `/api/reviews/transactions/{tx_id}/status` | 평가 가능 여부 조회 |

---

## 개발 시 테스트 방법

평가 화면까지 빠르게 확인하려면 Supabase Table Editor에서  
`transactions` 테이블의 해당 row `status`를 `settled`로 직접 수정하면 됩니다.  
그 후 문의 상세 페이지를 열면 평가 섹션이 바로 노출됩니다.
