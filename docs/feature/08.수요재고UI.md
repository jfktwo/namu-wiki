# 수요·재고 UI 명세

등록 화면과 수정 화면의 필드 구성, 그리고 목록 카드의 표시 항목을 동일한 기준으로 관리한다.

---

## 원칙

- **등록 = 수정**: 등록 폼에 있는 모든 필드는 수정 폼에도 동일하게 존재해야 한다.
- **카드 표시 = 핵심 규격 중심**: 목록 카드에는 거래 판단에 필요한 핵심 항목만 표시한다. 특히 근원직경(`root_diameter`)은 조경수 거래의 기준 규격으로 반드시 표시한다.

---

## 수요 (Demand)

### 등록/수정 공통 필드

| 섹션 | 필드 | 비고 |
|------|------|------|
| 기본 | 수종 (`species`) | SpeciesSelect 드롭다운 |
| 규격 | 근원직경 (`root_diameter`) | ★ 거래 기준, cm |
| 규격 | 흉고직경 (`breast_diameter`) | 교목용, cm |
| 규격 | 수고 (`height`) | m |
| 규격 | 수관폭 (`width`) | m |
| 규격 | 수령 (`age_year`) | 년 |
| 수량 | 수량 (`quantity`) | 그루, 필수 |
| 수량 | 필요 시기 (`needed_at`) | 날짜, 필수 |
| 추가 | 긴급 여부 (`is_urgent`) | 체크박스 |
| 추가 | 수요 마감일 (`expires_at`) | 날짜, 선택 |
| 추가 | 현장 주소 (`site_address`) | 납품 위치와 다를 때, 선택 |
| 예산 | 총 최대 예산 (`budget`) | 원, 필수 |
| 예산 | 그루당 최대 단가 (`unit_price_max`) | 원, 선택 |
| 위치 | 납품 위치 (`lat`, `lng`, `address`) | 지도 피커, 필수 |
| 사진 | 참고 사진 (`photo_urls`) | 최대 3장, 선택 |
| 상태 | 상태 (`status`) | 수정 시만 표시: 공개·매칭있음·마감·숨김 |

> **규격 필드**는 `SpecInput` 컴포넌트로 입력하며, `buildSpecString()`으로 `spec` 문자열을 자동 생성한다.  
> 예: `5년생 R8 H2.5 W1.2`

### 수요 카드 (목록)

```
[사진/이니셜]  [공개] [긴급]
               왕벚나무 R8          ← 근원직경 강조
               100그루 · 예산 5,000,000원
               필요 시기: 2025-05-01
```

| 표시 항목 | 설명 |
|-----------|------|
| 상태 뱃지 | 공개 / 매칭있음 / 마감 / 숨김 |
| 긴급 뱃지 | `is_urgent = true` 일 때만 표시 |
| 수종 + 근원직경 | `왕벚나무 R8` 형태, 근원직경 없으면 수종만 |
| 수량 + 예산 | `100그루 · 예산 5,000,000원` |
| 필요 시기 | `needed_at` |

## 재고 (Stock)

### 등록/수정 공통 필드

| 섹션 | 필드 | 비고 |
|------|------|------|
| 기본 | 수종 (`species`) | SpeciesSelect 드롭다운 |
| 규격 | 근원직경 (`root_diameter`) | ★ 거래 기준, cm |
| 규격 | 흉고직경 (`breast_diameter`) | 교목용, cm |
| 규격 | 수고 (`height`) | m |
| 규격 | 수관폭 (`width`) | m |
| 규격 | 수령 (`age_year`) | 년 |
| 수량 | 공급 가능 수량 (`quantity`) | 그루, 필수 |
| 수량 | 최소 주문 수량 (`min_order_qty`) | 그루, 선택 |
| 가격 | 단가 (`unit_price`) | 원/그루, 필수 |
| 가격 | 가격 협상 가능 (`price_negotiable`) | 체크박스 |
| 서비스 | 굴취 가능 (`can_dig`) | 토글 버튼 |
| 서비스 | 운송 가능 (`can_transport`) | 토글 버튼 |
| 서비스 | 식재 가능 (`can_plant`) | 토글 버튼 |
| 기간 | 유효기간 (`expires_at`) | 날짜, 필수 |
| 농장 | 농장 주소 (`farm_address`) | 선택 |
| 농장 | 야적장 주소 (`yard_address`) | 선택 |
| 농장 | 수목 위치 메모 (`location_memo`) | 선택 |
| 위치 | 위치 (`lat`, `lng`, `address`) | 지도 피커, 필수 |
| 사진 | 사진 (`photo_urls`) | 최대 5장, **최소 1장 필수** |
| 상태 | 상태 (`status`) | 수정 시만 표시: 활성·숨김·유효기간만료 |

### 재고 카드 (목록 그리드)

```
┌────────────────┐
│    [사진]      │
│ [활성]         │
├────────────────┤
│ 왕벚나무       │
│ R8 · 100그루   │  ← 근원직경 + 수량
│    최소 10그루 │
│ 50,000원/그루  │  ← 협상가능 표시
│ [굴취][운송]   │  ← 서비스 뱃지
└────────────────┘
```

| 표시 항목 | 설명 |
|-----------|------|
| 상태 뱃지 | 활성 / 숨김 / 유효기간만료 (사진 위 오버레이) |
| 수종 | 상단 굵게 |
| 근원직경 + 수량 | `R8 · 100그루`, 근원직경 없으면 수량만 |
| 최소 주문 수량 | `최소 N그루`, `min_order_qty` 있을 때만 |
| 단가 | `50,000원/그루` + `협상가능` 레이블 |
| 서비스 뱃지 | 굴취·운송·식재 활성화된 항목만 표시 |

## 컴포넌트 구조

```
SpeciesSelect   수종 선택 드롭다운 (shared)
SpecInput       규격 입력 (근원직경·흉고직경·수고·수관폭·수령) (shared)
buildSpecString spec 문자열 자동 생성 (shared)
LocationPicker  지도 위치 선택 (shared)
ImageUpload     사진 업로드 (shared)
```

수정 폼은 기존 데이터를 로드할 때 `root_diameter`, `breast_diameter`, `height`, `width`, `age_year` 컬럼 값을 직접 `SpecValue` 객체에 매핑한다 (spec 문자열 파싱 불필요).

```typescript
setSpec({
  root_diameter:   s.root_diameter,
  breast_diameter: s.breast_diameter,
  height:          s.height,
  width:           s.width,
  age_year:        s.age_year,
})
```
