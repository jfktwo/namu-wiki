# 기능 발전 체크리스트

나무 장터를 MVP에서 실제 운영 가능한 서비스로 키우기 위한 기능 목록이다.  
작업할 때는 각 항목을 하나씩 체크하면서 진행한다.

## 현재 기준

- 서비스 성격: 수요자와 공급자를 연결하는 조경수 B2B 매칭 플랫폼
- 현재 핵심 흐름: 회원가입/로그인 -> 수요 등록 -> 공급 재고 등록 -> 매칭 -> 문의 요청
- 현재 데이터 기준: 수요, 재고, 매칭, 문의, 샘플 이미지가 들어간 시험 환경
- 추천 배포 구조: 프론트엔드 Vercel, 백엔드 FastAPI 서버, DB/Storage Supabase, 모바일은 Capacitor 패키징

## 진행 메모

- 스키마 변경은 반드시 migration 파일과 백엔드 schema, 프론트엔드 type을 같이 수정한다.
- 수익화 기능은 결제 연동 전에 먼저 무료/유료 권한 제한 로직부터 만든다.
- 앱 패키징은 웹 배포가 안정화된 뒤 진행하는 것이 좋다.
- 이미지는 Supabase 무료 구간 안에서는 충분하지만, 운영 전에는 압축과 삭제 정책을 정해야 한다.
