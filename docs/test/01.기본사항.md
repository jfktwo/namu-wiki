# 01. 테스트 기본사항

## 테스트 구조

```
프로젝트 루트/
├── tests/
│   └── test_api.py          # 백엔드 API 통합 테스트 (pytest)
└── frontend/
    └── tests/
        └── inquiries.model.test.ts   # 프론트엔드 유닛 테스트 (vitest)
```

---

## 백엔드 테스트 (pytest)

### 환경 준비
```bash
# 백엔드 서버가 실행 중이어야 함
cd backend
uvicorn app.main:app --reload
```

### 실행 방법
```bash
# 프로젝트 루트에서
cd backend && ../.venv/Scripts/pytest ../tests/test_api.py -v
```

### 필요 패키지
```bash
python -m pip install pytest httpx
```

### 동작 방식
- 실제 백엔드 서버(`http://localhost:8000`)에 HTTP 요청을 보내는 **통합 테스트**
- 매 실행마다 랜덤 suffix로 고유 이메일 생성 → DB에 실제 데이터 쌓임
- `scope="session"` fixture로 buyer/seller 토큰을 세션 내 공유

---

## 프론트엔드 테스트 (vitest)

### 환경 준비
```bash
cd frontend
npm install
```

### 실행 방법
```bash
cd frontend
npx vitest run            # 1회 실행
npx vitest                # watch 모드
npx vitest --coverage     # 커버리지 포함
```

### 동작 방식
- 서버 없이 실행 가능한 **유닛 테스트**
- `@/` 경로 alias는 `vite.config.ts` 설정을 vitest가 공유해서 사용

---

## 주의사항

- 백엔드 테스트는 **Supabase에 실제 데이터가 생성됨** — 테스트 전용 DB 분리 권장 (future 작업)
- 테스트 실행 후 생성된 유저/재고/수요는 수동 정리 필요
