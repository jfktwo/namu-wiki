## 추가로 작성이 필요한 테스트

### 백엔드 (pytest)

| 영역 | 항목 | 우선순위 |
|------|------|---------|
| 수요 | 수요 단건 조회, 수정, 삭제 | 높음 |
| 재고 | 재고 단건 조회, 수정, 삭제 | 높음 |
| 어드민 | admin이 타인 재고/수요 수정·삭제 가능 확인 | 높음 |
| 권한 | 비로그인 상태로 API 접근 → 401 | 높음 |
| 문의 | 문의 생성, 목록 조회, 중복 문의 방지 | 중간 |
| 매칭 | 수요+재고 등록 후 매칭 자동 생성 확인 | 중간 |
| 매칭 | 단건 매칭 조회 | 중간 |
| 인증 | 만료된 토큰으로 접근 → 401 | 중간 |
| 파일 | 이미지 업로드 성공 / 허용되지 않는 형식 → 400 | 낮음 |
| 유저 | 어드민 전용 유저 목록 조회 | 낮음 |

### 프론트엔드 (vitest)

| 영역 | 항목 | 우선순위 |
|------|------|---------|
| 유틸 | 날짜 포맷 함수 | 중간 |
| 유틸 | 가격 포맷 함수 (toLocaleString) | 낮음 |
| Auth | useAuth hook — 토큰 저장/로드 | 높음 |
| 매칭 | 거리 필터링 로직 | 중간 |

## 향후 개선사항

- **테스트 전용 DB 분리** — 현재 테스트 실행 시 실제 Supabase에 데이터 생성됨
- **CI 연동** — GitHub Actions에서 자동 실행 (배포 전 검증)
- **E2E 테스트** — Playwright로 실제 브라우저 시나리오 테스트 (로그인 → 재고 등록 → 매칭 확인)
