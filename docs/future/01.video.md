# 01. 동영상 업로드 기능

**상태:** 미구현 (추후 작업 예정)

---

## 배경

Supabase Storage 무료 플랜은 1GB 제한이라 동영상 저장에 부적합.
이미지는 Supabase 유지, 동영상은 Cloudflare R2를 별도로 사용하기로 결정.

---

## 결정된 방식: Cloudflare R2

- 무료 10GB 저장 + **egress(전송) 비용 0원** (Google Cloud는 $0.12/GB)
- S3 API 완벽 호환 → boto3로 연동 가능
- 브라우저 → R2 직접 업로드 (presigned URL 방식, 백엔드 서버 부하 없음)

### 비교 검토했던 옵션들

| 방식 | 비용 | 결론 |
|------|------|------|
| Supabase Storage | 무료 1GB → 초과 $0.021/GB | 동영상엔 부족 |
| YouTube URL 붙여넣기 | 무료 | UX 불편 (사용자가 직접 업로드해야 함) |
| Google Cloud Storage | 저장 $0.020/GB + 전송 $0.12/GB | 전송 비용 과다 |
| **Cloudflare R2** | 저장 무료 10GB + 전송 0원 | **채택** |

---

## Cloudflare 사전 설정 (계정 생성 후)

1. cloudflare.com → **R2 Object Storage** → Create Bucket
   - 버킷 이름 예시: `namu-videos`
2. 버킷 → Settings → **Public Access → Allow Access**
   - 퍼블릭 URL 발급됨: `https://pub-xxxxx.r2.dev`
3. **My Profile → API Tokens → Create Token**
   - R2:Edit 권한 선택
   - Account ID / Access Key ID / Secret Access Key 복사
4. 프로젝트 `.env`에 추가:
```env
CF_ACCOUNT_ID=xxxx
CF_R2_ACCESS_KEY=xxxx
CF_R2_SECRET_KEY=xxxx
CF_R2_BUCKET=namu-videos
CF_R2_PUBLIC_URL=https://pub-xxxxx.r2.dev
```

## 구현 계획

### 백엔드
- `boto3` 패키지 추가 (S3 호환)
- `/api/videos/upload-url` 엔드포인트 — presigned URL 생성 반환
- `/api/videos/delete` 엔드포인트 — R2 파일 삭제

### 프론트엔드
- `VideoUpload` 컴포넌트
  - 갤러리에서 동영상 선택
  - 업로드 진행바 표시
  - 완료 후 URL 저장
- 재고(Stock) / 수요(Demand) 등록·수정 폼에 추가
- 상세 페이지에 video player 표시

### DB 마이그레이션
- `stocks` 테이블: `video_urls text[]` 컬럼 추가
- `demands` 테이블: `video_urls text[]` 컬럼 추가

## 부록: Cloudflare Pages 배포도 논의됨

Cloudflare R2 설정 중 Pages(정적 호스팅) 서비스도 확인됨.

| 서비스 | 용도 | 비용 |
|--------|------|------|
| Cloudflare Pages | 프론트엔드 호스팅 | 무료 (무제한 대역폭) |
| Cloudflare R2 | 파일 저장 | 무료 10GB |
| Cloudflare Workers | 서버리스 함수 | Python 지원 베타 → 불안정 |

→ FastAPI 백엔드는 Workers에서 실행 불가(베타), Railway/Render 별도 필요.
→ 배포 구성도 추후 작업 예정. (참고: [future/readme.md](readme.md))
