# 워크플로우

프로젝트를 진행할 때 필요한 역할, 작업 흐름, 산출물을 정리한다.

이 문서는 실제 사람이 맡아도 되고, Codex 같은 에이전트에게 역할을 나눠 맡길 때도 기준으로 사용할 수 있다.

---

## 기본 진행 흐름

```txt
기획
  -> 화면/UX 설계
  -> 기술 설계
  -> 프론트엔드 구현
  -> 백엔드 구현
  -> 연동 테스트
  -> QA
  -> 배포
  -> 운영 피드백 반영
```

---

| 아키텍트 | 프론트엔드/백엔드/DB/API 구조 결정 | 시스템 구조 문서, API 설계 방향, 기술 선택 기준 |

## 에이전트별 필요 스킬

### 디렉터(PM)

- 요구사항 우선순위 판단
- MVP 범위 조절
- 일정과 마일스톤 관리
- 작업 분해와 담당자 배정
- 리스크 식별과 의사결정 정리
- 회의록, 결정사항, 변경사항 기록

### 기획자

- 사용자 페르소나 정의
- 사용자 시나리오 작성
- 기능 요구사항 정리
- 업무 흐름과 예외 흐름 분석
- 정책, 상태값, 권한 규칙 정의
- 화면별 입력 항목과 검증 조건 정리

### 디자이너

- 정보 구조 설계
- 와이어프레임 작성
- 모바일/데스크톱 반응형 화면 설계
- 폼, 목록, 상세, 검색, 필터 UX 설계
- 공통 UI 컴포넌트 규칙 정의
- 접근성, 가독성, 사용성 검토

### 아키텍트

- 프론트엔드/백엔드/DB 책임 분리
- API 계약 설계
- 데이터 흐름 설계
- 폴더 구조와 모듈 경계 설계
- 확장성과 유지보수성 검토
- 기술 선택 근거 정리

### 프론트엔드 개발자

- React 컴포넌트 설계
- TypeScript 타입 모델링
- Vite 개발환경 관리
- API 연동과 에러 상태 처리
- 폼 입력, 목록, 상세 화면 구현
- 반응형 CSS 작성
- 프론트엔드 빌드와 린트 검증

### 백엔드 개발자

- FastAPI 라우터 구현
- Pydantic 스키마 작성
- 서비스 로직과 repository 분리
- 입력 검증과 예외 처리
- 인증/권한 처리
- Supabase, Kakao Map 등 외부 API 연동
- API 문서와 응답 형식 관리

### 데이터베이스 담당

- PostgreSQL 테이블 설계
- Supabase 프로젝트 구조 이해
- 마이그레이션 작성과 적용
- seed 데이터 관리
- 인덱스와 조회 성능 검토
- 데이터 무결성, 제약조건, 관계 설계
- 백업과 복구 기준 정리

### DevOps/환경 담당

- 로컬 개발환경 구성
- `.env`와 `.env.example` 관리
- 실행 스크립트와 포트 규칙 정리
- Git, GitHub, 브랜치 흐름 관리
- 배포 환경 변수 분리
- 빌드/실행 오류 원인 분석
- 개발환경 문서화

### 테스터(QA)

- 테스트 케이스 작성
- 정상/예외/경계값 검증
- 프론트엔드 화면 검증
- API 응답 검증
- 회귀 테스트 수행
- 버그 재현 단계 작성
- 완료 기준 충족 여부 확인

### 릴리즈 관리자

- 릴리즈 체크리스트 작성
- 빌드 산출물 확인
- 배포 전 환경 변수 점검
- 변경사항과 영향 범위 정리
- 릴리즈 노트 작성
- 배포 후 스모크 테스트 관리
- 롤백 기준 정리

## 역할별 판단 기준

### 디렉터(PM)

- 지금 반드시 해야 하는 일과 나중에 해도 되는 일을 나눈다.
- 기능 범위가 커질 때 MVP 기준으로 줄인다.
- 작업 순서를 정하고 막힌 지점을 정리한다.

### 기획자

- 사용자가 누구인지 정의한다.
- 수요자와 공급자의 주요 행동을 시나리오로 만든다.
- 기능 요구사항을 개발 가능한 단위로 나눈다.

### 디자이너

- 목록, 등록, 상세, 매칭 결과 화면의 흐름을 잡는다.
- 모바일과 데스크톱에서 모두 사용할 수 있는 화면 구조를 만든다.
- 버튼, 입력폼, 상태 표시 같은 공통 UI 규칙을 정리한다.

### 아키텍트

- 프론트엔드, 백엔드, 데이터베이스의 책임을 나눈다.
- API 경계와 데이터 흐름을 정리한다.
- 기술 선택이 현재 MVP와 이후 확장에 맞는지 검토한다.

### 프론트엔드 개발자

- React + Vite + TypeScript 기반 화면을 구현한다.
- `pages`, `features`, `shared` 구조를 유지한다.
- 백엔드 API 응답 타입을 기준으로 화면을 구성한다.

### 백엔드 개발자

- FastAPI 기반 API를 구현한다.
- 입력 검증, 비즈니스 로직, DB 접근을 분리한다.
- Supabase, Kakao Map 같은 외부 연동을 백엔드에서 관리한다.

### 데이터베이스 담당

- 수요, 재고, 매칭, 알림 테이블 구조를 설계한다.
- 마이그레이션 파일과 seed 데이터를 관리한다.
- 조회 성능과 데이터 무결성을 확인한다.

### DevOps/환경 담당

- 로컬 실행 방법을 문서화한다.
- `.env`, `.env.example`, 실행 스크립트, 포트 규칙을 정리한다.
- 배포 환경에서 필요한 설정을 미리 분리한다.

### 테스터(QA)

- 기능별 정상/예외 케이스를 체크리스트로 만든다.
- 프론트엔드와 백엔드 연동을 검증한다.
- 발견한 문제를 재현 단계와 기대 결과로 기록한다.

### 릴리즈 관리자

- 릴리즈 전 빌드, 테스트, 환경 변수, 문서 상태를 확인한다.
- 변경사항을 릴리즈 노트로 정리한다.
- 배포 후 확인해야 할 항목을 체크한다.

## 완료 기준

작업은 아래 조건을 만족해야 완료로 본다.

- 요구사항이 문서 또는 이슈에 정리되어 있다.
- 구현 파일과 설정 파일이 예상 위치에 있다.
- 실행 또는 빌드 명령이 성공한다.
- 테스트 또는 수동 검증 결과가 기록되어 있다.
- 다음 작업자가 이어받을 수 있도록 남은 일이 정리되어 있다.
