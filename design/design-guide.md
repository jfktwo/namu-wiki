# 디자인 가이드

나무 장터를 아름다운 앱으로 바꾸기 위한 기준 문서다.  
목표는 화려한 쇼핑몰이 아니라, 조경 현장에서 신뢰하고 반복해서 쓰는 B2B 거래 앱이다.

## 1. 디자인 방향

### 한 문장 정의

> 조경 현장에서 바로 쓸 수 있는, 깨끗하고 신뢰감 있는 초록 계열 B2B 거래 앱

### 사용자가 느껴야 하는 인상

- 신뢰감 있다.
- 현장 업무에 맞게 빠르다.
- 나무 실물 사진이 잘 보인다.
- 정보가 정돈되어 비교하기 쉽다.
- 너무 장식적이지 않다.
- 공급자와 수요자가 거래를 시작하기 편하다.

### 피해야 할 인상

- 초록색이 너무 많아서 촌스러운 화면
- 카드와 그림자가 과해서 장터보다 광고 페이지처럼 보이는 화면
- 사진보다 버튼과 장식이 더 눈에 띄는 화면
- 수종, 규격, 거리, 가격을 한눈에 비교하기 어려운 화면
- 모바일에서 버튼과 텍스트가 답답하게 붙어 있는 화면

## 2. 핵심 원칙

### 2.1 사진이 첫 번째다

이 앱의 핵심 상품은 나무다. 사용자는 설명보다 먼저 실물 사진을 보고 판단한다.

- 재고 카드에는 사진을 크게 배치한다.
- 수요 카드도 참고 사진이 있으면 반드시 보여준다.
- 사진이 없을 때는 빈 박스가 아니라 깔끔한 기본 이미지 영역을 보여준다.
- 사진 위에 텍스트를 과하게 올리지 않는다.
- 대표 사진 1장을 크게, 나머지 사진은 썸네일로 보여주는 구조가 좋다.

### 2.2 정보 순서가 일정해야 한다

카드마다 정보 순서가 달라지면 비교가 어려워진다.

추천 순서:

1. 사진
2. 수종명
3. 규격
4. 수량
5. 위치/거리
6. 가격/예산
7. 상태
8. 주요 액션

### 2.3 초록은 포인트로 쓴다

초록색은 브랜드의 중심이지만 화면 전체를 덮으면 피로해진다.

초록색 사용 위치:

- 주요 버튼
- 선택된 탭
- 매칭 가능 상태
- 거리/추천 점수 강조
- 작은 점 패턴
- 성공/활성 상태

초록색을 피할 위치:

- 긴 설명문 배경
- 모든 카드 테두리
- 오류 상태
- 삭제 버튼
- 화면 전체 배경

### 2.4 업무용 앱은 정돈감이 아름다움이다

나무 장터는 감성 앱보다 거래 도구에 가깝다.

- 큰 장식보다 정렬, 여백, 정보 밀도가 중요하다.
- 화면마다 같은 위치에 같은 기능이 있어야 한다.
- 버튼 이름은 짧고 분명해야 한다.
- 목록은 스캔하기 쉬워야 한다.
- 상세 화면은 거래 판단에 필요한 정보를 빠짐없이 보여줘야 한다.

## 3. 시각 시스템

## 3.1 컬러

기본 팔레트:

| 용도 | 색상 | 사용처 |
|---|---|---|
| Primary Green | `#1A6B4A` | 주요 버튼, 선택 상태, 브랜드 강조 |
| Fresh Green | `#2D9B6A` | hover, 보조 강조, 진행 상태 |
| Leaf Green | `#4CAF6A` | 작은 점, 매칭 가능 표시 |
| Soft Green | `#EAF6EF` | 선택된 배지 배경, 연한 상태 표시 |
| Background | `#F8FAF8` | 전체 배경 |
| Surface | `#FFFFFF` | 카드, 패널, 입력 영역 |
| Border | `#E8EDE8` | 카드/입력/목록 구분선 |
| Text Strong | `#1A1A2E` | 제목, 핵심 정보 |
| Text Muted | `#6B7280` | 보조 설명, 메타 정보 |
| Warning | `#F59E0B` | 마감 임박, 확인 필요 |
| Danger | `#EF4444` | 삭제, 오류, 차단 |

운영 규칙:

- 한 화면의 주인공은 초록색이 아니라 콘텐츠다.
- 배경은 거의 흰색에 가까운 연회색을 쓴다.
- 카드 내부는 흰색을 기본으로 한다.
- 강조가 필요한 곳만 초록색을 사용한다.
- 그라데이션은 사용하더라도 주요 버튼 1종에만 제한한다.

## 3.2 초록색 점 패턴

초록색 점은 이 앱의 브랜드 신호로 사용한다.  
의미는 나무, 위치, 매칭 포인트, 생장점이다.

사용하기 좋은 곳:

- 로고 옆 작은 마크
- 로그인/회원가입 배경의 낮은 투명도 패턴
- 매칭 가능 배지
- 선택된 필터 옆 점
- 진행 단계 표시
- 빈 화면의 작은 장식

주의할 점:

- 점 패턴을 화면 전체에 강하게 깔지 않는다.
- 카드 안에 반복해서 넣지 않는다.
- 점 크기는 4px~8px를 기본으로 한다.
- 중요한 숫자나 버튼 근처에 너무 많이 배치하지 않는다.

## 3.3 모서리와 그림자

현재 프론트에는 `rounded-xl`, `rounded-2xl` 느낌이 많다.  
운영용 B2B 앱에서는 조금 더 차분한 반경이 좋다.

권장:

- 카드: 8px
- 입력 필드: 8px
- 버튼: 8px
- 배지: 999px
- 모달: 12px

그림자:

- 기본 카드는 강한 그림자보다 얇은 테두리를 우선한다.
- hover 그림자는 살짝만 올린다.
- 모바일에서는 hover 효과보다 눌림 상태가 중요하다.

## 3.4 타이포그래피

권장 폰트:

- 한글: `Noto Sans KR`
- 숫자/영문: 시스템 기본 또는 `Inter`

크기 기준:

| 용도 | 크기 |
|---|---:|
| 페이지 제목 | 24px~28px |
| 섹션 제목 | 18px~22px |
| 카드 제목 | 16px~18px |
| 본문 | 14px~16px |
| 메타 정보 | 12px~13px |

규칙:

- 업무 화면 안에서 너무 큰 히어로 타이포를 쓰지 않는다.
- 카드 안 제목은 짧고 굵게 쓴다.
- 수량, 거리, 가격 같은 숫자는 한눈에 보이게 굵게 처리한다.
- 글자 간격은 기본값을 유지한다.

## 4. 화면별 가이드

## 4.1 로그인/회원가입

목표:

- 사용자가 빠르게 들어오게 한다.
- 앱의 정체성을 처음에 보여준다.
- 역할 선택을 명확하게 한다.

구성:

- 좌측 또는 상단: 브랜드명, 짧은 설명
- 중앙: 로그인/회원가입 폼
- 배경: 매우 연한 초록 점 패턴
- 역할 선택: 수요자 / 공급자 / 관리자 카드

주의:

- 로그인 화면에 설명을 너무 많이 넣지 않는다.
- 회원가입 역할 선택은 버튼보다 카드형 선택이 좋다.
- 모바일에서는 폼이 첫 화면에 바로 보여야 한다.

## 4.2 수요 목록

목표:

- 내가 등록한 수요와 매칭 상태를 빠르게 확인한다.

카드 정보 순서:

1. 수종
2. 규격
3. 필요 수량
4. 납품 주소
5. 희망 시기
6. 예산
7. 매칭 후보 수

강조:

- 매칭 후보가 있으면 초록 배지로 표시한다.
- 마감 임박은 주황색으로 표시한다.
- 취소/종료 상태는 회색으로 낮춘다.

## 4.3 재고 목록

목표:

- 공급자가 자신의 재고 상태를 빠르게 관리한다.
- 수요자는 재고 사진과 조건을 보고 문의할지 판단한다.

카드 정보 순서:

1. 대표 사진
2. 수종
3. 규격
4. 보유 수량
5. 단가
6. 위치
7. 운송/굴취/식재 가능 여부
8. 문의 또는 수정 버튼

강조:

- 사진 비중을 높인다.
- 가격 협의 가능 상태를 배지로 표시한다.
- 판매 완료/숨김 상태는 카드 톤을 낮춘다.

## 4.4 매칭 목록

목표:

- 수요와 재고가 왜 매칭됐는지 설명한다.

필수 표시:

- 거리
- 수종 일치 여부
- 규격 일치 여부
- 수량 충족 여부
- 가격 조건
- 추천 점수

표현:

- 단순히 “32km”만 보여주지 말고 “32km 이내”처럼 거래 맥락을 준다.
- 조건 통과 항목은 작은 초록 체크/배지로 보여준다.
- 부족한 조건은 회색 또는 주황색으로 보여준다.

## 4.5 문의/견적

목표:

- 문의를 거래 상담으로 이어지게 한다.

목록:

- 보낸 문의 / 받은 문의 탭
- 상태 배지: 요청, 답변, 견적, 수락, 거절
- 마지막 메시지 시간
- 연결된 수요/재고 요약

상세:

- 상단에 수요/재고 요약
- 중간에 메시지 흐름
- 하단에 답변/견적 작성 영역

주의:

- “이미 문의를 보냈거나 오류가 발생했습니다”처럼 원인을 합친 문구는 피한다.
- 사용자가 다음에 무엇을 해야 하는지 알 수 있어야 한다.

## 4.6 관리자

목표:

- 예쁘기보다 빠르고 정확하게 관리한다.

구성:

- 목록은 테이블 중심
- 상태 필터는 상단 고정
- 숨김/차단/삭제 같은 위험 액션은 분리
- 통계는 대시보드 상단에 작게 배치

주의:

- 관리자 화면은 장식보다 밀도가 중요하다.
- 위험 액션은 확인 모달을 사용한다.

## 5. 컴포넌트 가이드

## 5.1 버튼

종류:

| 종류 | 사용처 |
|---|---|
| Primary | 등록, 저장, 문의 요청, 견적 보내기 |
| Secondary | 취소, 뒤로가기, 필터 초기화 |
| Ghost | 보조 이동, 상세 보기 |
| Danger | 삭제, 숨김, 차단 |

규칙:

- 한 화면의 Primary 버튼은 1개를 기본으로 한다.
- Primary 버튼은 초록색을 사용한다.
- Danger 버튼에는 초록색을 쓰지 않는다.
- 아이콘이 자연스러운 기능은 아이콘을 같이 쓴다.

## 5.2 카드

카드는 반복 항목에만 사용한다.

좋은 카드:

- 재고 카드
- 수요 카드
- 매칭 카드
- 문의 카드

피해야 할 카드:

- 페이지 전체를 감싸는 큰 카드
- 카드 안의 카드
- 단순 섹션을 카드처럼 띄우는 디자인

## 5.3 배지

배지는 상태를 짧게 보여주는 요소다.

예시:

- 매칭 있음
- 답변 대기
- 견적 도착
- 판매 가능
- 협의 가능
- 마감 임박
- 숨김

규칙:

- 상태별 색을 고정한다.
- 색만으로 의미를 전달하지 않고 텍스트를 함께 쓴다.
- 배지는 짧게 쓴다.

## 5.4 입력 폼

규칙:

- 필수 항목은 명확하게 표시한다.
- 숫자 입력에는 단위를 같이 보여준다.
- 주소 입력 뒤에는 좌표 변환 상태를 보여준다.
- 사진 업로드는 미리보기가 있어야 한다.
- 오류 메시지는 입력 필드 바로 아래에 둔다.

## 6. 모바일 가이드

모바일은 이 앱에서 중요하다. 현장에서는 PC보다 휴대폰으로 확인할 가능성이 높다.

권장 구조:

- 하단 탭: 홈, 수요, 재고, 매칭, 문의
- 목록 카드는 1열
- 필터는 상단 접힘 영역
- 주요 버튼은 화면 하단 근처
- 상세 화면은 사진 -> 핵심 정보 -> 조건 -> 문의 순서

터치 기준:

- 버튼 높이 최소 44px
- 카드 간 간격 최소 12px
- 필터 칩은 손가락으로 누르기 쉽게 만든다.
- 작은 텍스트 링크보다 버튼/아이콘 버튼을 우선한다.

## 7. 아름다움 체크리스트

화면을 고칠 때마다 아래를 확인한다.

- [ ] 이 화면에서 사용자가 가장 먼저 봐야 할 것이 분명한가?
- [ ] 사진이 충분히 크게 보이는가?
- [ ] 수종, 규격, 수량, 거리, 가격을 빠르게 비교할 수 있는가?
- [ ] 초록색이 너무 많이 쓰이지 않았는가?
- [ ] 버튼이 너무 많아 보이지 않는가?
- [ ] 모바일에서 터치하기 편한가?
- [ ] 카드 안 텍스트가 넘치지 않는가?
- [ ] 상태 색상과 문구가 일관적인가?
- [ ] 빈 상태와 오류 상태가 자연스러운가?
- [ ] 같은 기능은 같은 위치에 있는가?

## 8. 현재 프론트 개선 메모

현재 코드 기준으로 다음 항목을 먼저 정리하면 앱 인상이 빠르게 좋아진다.

- [ ] `Card`의 `rounded-2xl`을 8px 기준으로 조정
- [ ] 카드 그림자를 약하게 하고 테두리 중심으로 정리
- [ ] `Button`의 강한 그라데이션을 줄이고 단색 Primary로 정리
- [ ] 재고/수요/매칭 카드의 정보 순서 통일
- [ ] 사진 영역 비중 확대
- [ ] 상태 배지 색상 체계 통일
- [ ] 모바일 하단 탭 구조 검토
- [ ] 오류/빈 상태 문구 정리
- [ ] 입력 폼의 단위 표시 정리
- [ ] 초록 점 패턴을 로그인/빈 화면/브랜드 영역에만 제한적으로 사용

## 9. 우선 적용 순서

1. 공통 컴포넌트 기준 정리: Button, Card, Badge, Input
2. 재고 목록 카드 개선
3. 매칭 목록 카드 개선
4. 수요 목록 카드 개선
5. 문의 목록/상세 개선
6. 로그인/회원가입 화면 개선
7. 관리자 화면 정리
