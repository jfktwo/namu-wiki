# 구성요소별 디자인 개선안

생성 시각: 2026-06-14T05:37:04.531Z

## 컴포넌트 진단

| 구성 요소 | 현재 상태 | 문제 | 개선 방향 |
|---|---|---|---|
| 레이아웃 | 모바일 앱처럼 최대 폭이 제한된 중앙 컬럼과 하단 탭 구조를 사용한다. | 데스크톱에서도 모바일 폭 중심이라 넓은 화면의 정보 밀도를 활용하지 못한다. | 모바일은 현재 구조를 유지하고, 데스크톱은 2열 목록/필터/상세 패널 구조를 검토한다. |
| 카드 | rounded-xl/2xl, shadow-md/shadow-xl 계열의 부드러운 카드가 많다. | B2B 거래 도구보다는 감성 모바일 앱에 가까워 보일 수 있다. | 반복 항목 카드만 유지하고 반경 8px, 얇은 border, 낮은 shadow로 정리한다. |
| 버튼 | 초록 그라데이션과 큰 라운드 버튼이 주요 액션에 사용된다. | 강한 시각 효과가 많아지면 콘텐츠보다 버튼이 먼저 보인다. | Primary는 단색 초록, Secondary는 흰색+테두리, Danger는 붉은 계열로 통일한다. |
| 사진 | 재고/수요 목록에서 이미지가 핵심 자산이지만 화면별 비중 점검이 필요하다. | 사진보다 텍스트/상태 UI가 먼저 보이면 거래 판단력이 떨어진다. | 재고 카드는 대표 사진을 크게, 수요 카드는 참고 사진이 있을 때 상단 썸네일을 노출한다. |
| 상태 배지 | 상태 표시가 화면별로 분산되어 있다. | 매칭 있음, 답변 대기, 협의 가능, 마감 임박 등의 색상 체계가 흔들릴 수 있다. | 상태별 색상과 문구를 고정하고 색만이 아니라 텍스트로 의미를 함께 전달한다. |
| 한글 문구 | 일부 파일/화면에 깨진 한글 문자열이 남아 있다. | 디자인 완성도 이전에 신뢰감을 크게 떨어뜨린다. | 디자인 개편 1순위로 화면 문구를 정상 한글로 복구한다. |

## 우선순위

1. 화면 문구 정상화
2. Button, Card, Badge, Input 공통 컴포넌트 재정의
3. 재고 카드 사진 중심 구조로 변경
4. 매칭 카드에 추천 사유/점수 구조 추가
5. 수요 카드 정보 순서 통일
6. 문의/견적 상태 배지 체계 통일
7. 관리자 화면은 모바일 카드보다 테이블/필터 중심으로 분리

## 적용 전 체크

- [ ] 공통 디자인 토큰을 `frontend/src/index.css`에 정리한다.
- [ ] `Button`의 그라데이션 강도를 줄인다.
- [ ] `Card`의 radius/shadow 기준을 조정한다.
- [ ] 화면별 카드 정보 순서를 문서 기준과 맞춘다.
- [ ] 모바일과 데스크톱 화면을 모두 다시 캡쳐해 비교한다.
