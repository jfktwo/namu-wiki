# 05. 디자인 시안

나무 장터 MVP의 화면 디자인 방향을 정리한다.

핵심 모티프는 **초록색 점 패턴**이다. 나무, 묘목, 농장, 조경의 자연스러운 이미지를 살리되, 화면은 업무용 서비스처럼 깔끔하고 빠르게 읽히도록 만든다.

---

## 1. 디자인 목표

- 수요자와 공급자가 필요한 정보를 빠르게 찾을 수 있게 한다.
- 수목 거래 서비스의 자연스러운 느낌을 초록색 포인트로 표현한다.
- 목록, 필터, 상태, 매칭 정보를 한눈에 읽기 쉽게 만든다.
- 장식은 최소화하고, 초록색 점을 상태와 흐름을 알려주는 UI 요소로 사용한다.

---

## 2. 톤앤매너

| 항목 | 방향 |
|------|------|
| 전체 느낌 | 깔끔한 B2B 마켓, 조경/농장 서비스 감성 |
| 분위기 | 신뢰감, 실용성, 자연스러움 |
| 장식 | 작은 초록색 점, 얇은 선, 넓은 여백 |
| 화면 밀도 | 목록과 정보는 촘촘하게, 등록 화면은 여유 있게 |
| 카드 스타일 | 흰색 카드, 얇은 테두리, 8px 이하 둥근 모서리 |
| 버튼 스타일 | 명확한 액션 중심, 초록색은 주요 버튼에만 사용 |

| Deep Green | `#17633D` | 제목 강조, 활성 메뉴 |
| Soft Green | `#EAF6EF` | 선택 배경, 상태 배지 배경 |
| Leaf Dot | `#4CAF6A` | 초록색 점 패턴 |

| Danger | `#C2413B` | 오류, 삭제, 숨김 처리 |

## 4. 초록색 점 사용 규칙

초록색 점은 단순 장식이 아니라 상태와 흐름을 알려주는 신호로 사용한다.

### 사용 위치

- 로고 또는 서비스명 옆 포인트
- 목록 카드의 상태 표시
- 매칭 가능 여부 표시
- 선택된 필터 표시
- 단계 진행 표시
- 비어 있는 화면의 작은 배경 패턴

### 사용 예시

```txt
● 매칭 가능
● 50km 이내
● 예산 안
● 활성 재고
```

### 사용 제한

- 화면 전체를 점 패턴으로 덮지 않는다.
- 점은 4px~8px 크기를 기본으로 한다.
- 중요한 액션 버튼 안에는 점을 남발하지 않는다.
- 오류나 삭제 상태에는 초록색 점을 쓰지 않는다.

---

## 5. 레이아웃 원칙

### 공통 구조

```txt
상단 헤더
  - 서비스명
  - 현재 역할
  - 주요 메뉴
  - 사용자 메뉴

본문
  - 페이지 제목
  - 주요 액션 버튼
  - 필터/검색
  - 목록 또는 상세 정보
```

### 목록 화면

- 좌측 또는 상단에 검색/필터를 둔다.
- 목록은 카드형 또는 테이블형을 화면 성격에 맞게 사용한다.
- 수종, 규격, 수량, 지역, 가격, 상태를 카드 상단에서 바로 읽을 수 있게 한다.
- 매칭 가능한 항목은 초록 점과 배지로 강조한다.

### 등록 화면

- 입력 항목을 그룹으로 나눈다.
- 필수 입력값은 라벨과 보조 설명으로 명확히 표시한다.
- 주소 입력 후 좌표 변환 상태를 보여준다.
- 저장 버튼은 화면 하단 또는 우측 상단에 고정하지 않고 자연스럽게 배치한다.

## 6. 주요 화면 시안

### 6.1 로그인

목표:

- 사용자가 빠르게 로그인할 수 있게 한다.
- 서비스 정체성을 초록 점과 짧은 설명으로 보여준다.

구성:

```txt
나무 장터 ●
수요와 재고를 가까운 거리에서 연결합니다.

[이메일]
[비밀번호]
[로그인]

계정이 없나요? 회원가입
```

디자인 메모:

- 흰색 로그인 패널을 중앙에 둔다.
- 배경 한쪽에 아주 옅은 초록 점 패턴을 둔다.
- CTA 버튼은 Primary Green을 사용한다.

### 6.2 회원가입

목표:

- 가입 시 역할을 명확히 선택하게 한다.

구성:

```txt
역할 선택
[● 수요자] [○ 공급자] [○ 관리자]

기본 정보
[이메일]
[비밀번호]
[이름/업체명]
[전화번호]
[주요 지역]

[가입하기]
```

디자인 메모:

- 역할 선택은 카드형 선택 UI로 만든다.
- 선택된 역할 카드에 초록 점과 연한 초록 배경을 사용한다.

### 6.3 수요 목록

목표:

- 수요자가 본인 수요를 빠르게 확인하고 매칭 상태를 파악한다.

구성:

```txt
수요 목록                         [수요 등록]

[수종 검색] [지역] [상태]

● 매칭 있음
왕벚나무 / 3년생 / 50그루
납품지: 경기도 성남시
예산: 3,000,000원
매칭 후보 4건
```

디자인 메모:

- 매칭 후보가 있는 글은 초록 점과 `매칭 있음` 배지로 표시한다.
- 마감된 글은 흐린 회색 톤으로 표시한다.

### 6.4 수요 등록

목표:

- 필요한 나무 조건을 빠짐없이 입력하게 한다.

구성:

```txt
수요 등록

기본 정보
[수종] [수령/규격]
[수량] [필요 시기]

위치
[납품 주소]
● 좌표 변환 완료

예산
[총 예산]

[등록하기]
```

디자인 메모:

- 주소 입력 후 좌표 변환 상태를 초록 점으로 표시한다.
- 가격과 수량은 숫자 입력이므로 단위 표시를 붙인다.

### 6.5 재고 목록

목표:

- 공급자가 본인 재고 상태와 매칭된 수요를 확인한다.

구성:

```txt
재고 목록                         [재고 등록]

[수종 검색] [지역] [유효 상태]

● 활성
왕벚나무 / 3년생 / 80그루
위치: 충남 천안시
단가: 45,000원
매칭 수요 3건
```

디자인 메모:

- 유효기간 임박 재고는 Warning 색상을 사용한다.
- 활성 재고는 초록 점으로 표시한다.

### 6.6 재고 등록

목표:

- 공급자가 판매 가능한 재고 정보를 정확히 등록한다.

구성:

```txt
재고 등록

기본 정보
[수종] [수령/규격]
[수량] [단가]

위치
[재고 주소]
● 좌표 변환 완료

사진
[사진 업로드]

유효기간
[날짜 선택]

[등록하기]
```

디자인 메모:

- 사진 업로드 영역은 점선 테두리 박스로 표현한다.
- 등록 후 매칭 후보가 자동 계산된다는 안내를 짧게 보여준다.

### 6.7 매칭 목록

목표:

- 수요자와 공급자가 조건에 맞는 상대 후보를 빠르게 비교한다.

구성:

```txt
매칭 후보

● 32km 이내  ● 예산 안  ● 수량 충족

왕벚나무 / 3년생
수요: 50그루
재고: 80그루
예상 금액: 2,250,000원
거리: 32km

[문의 요청]
```

디자인 메모:

- 매칭 조건 통과 항목을 초록 점 리스트로 보여준다.
- 문의 요청 버튼은 주요 액션이므로 Primary Green을 사용한다.

### 6.8 문의 요청 목록

목표:

- 사용자가 보낸 문의와 받은 문의의 상태를 확인한다.

구성:

```txt
문의 요청

[보낸 문의] [받은 문의]

● 요청됨
왕벚나무 3년생 / 매칭 후보
상대: 천안 묘목장
상태: requested
```

디자인 메모:

- 상태별 배지를 사용한다.
- `requested`는 초록, `rejected`는 붉은 계열, `cancelled`는 회색을 사용한다.

### 6.9 관리자 게시글 관리

목표:

- 관리자가 문제 게시글을 빠르게 찾고 숨김 처리한다.

구성:

```txt
게시글 관리

[수요] [재고]
[검색] [상태]

왕벚나무 / 3년생 / 50그루
작성자: 경기 조경
상태: open

[숨김 처리]
```

디자인 메모:

- 관리자 화면은 장식보다 정보 밀도를 우선한다.
- 숨김 처리는 Danger 색상을 사용하고, 사유 입력 모달을 띄운다.

## 7. 공통 컴포넌트 규칙

### 버튼

| 종류 | 사용처 | 스타일 |
|------|--------|--------|
| Primary | 등록, 저장, 문의 요청 | 초록 배경, 흰색 텍스트 |
| Secondary | 취소, 뒤로가기 | 흰색 배경, 회색 테두리 |
| Danger | 숨김 처리, 삭제 | 붉은 계열 텍스트 또는 배경 |

### 배지

| 상태 | 색상 |
|------|------|
| 매칭 있음 | 초록 |
| 활성 | 초록 |
| 마감 | 회색 |
| 유효기간 임박 | 노랑/갈색 |
| 숨김 | 붉은 계열 |

### 입력 필드

- 라벨은 항상 표시한다.
- 필수값은 라벨 옆에 `필수` 배지를 붙인다.
- 숫자 입력은 단위를 함께 표시한다.
- 주소 입력은 좌표 변환 상태를 같이 표시한다.

### 카드

- 모서리는 8px 이하로 둥글게 한다.
- 카드 안에는 수종/규격/수량을 가장 먼저 배치한다.
- 상태 점은 카드 제목 왼쪽에 둔다.
- 상세 정보는 2열 그리드 또는 짧은 메타 텍스트로 정리한다.

---

## 8. 모바일 대응

- 헤더 메뉴는 하단 탭 또는 접이식 메뉴로 바꾼다.
- 목록 카드는 1열로 배치한다.
- 필터는 상단 접이식 패널로 둔다.
- 등록 화면은 한 줄에 하나의 입력만 배치한다.
- 문의 요청 버튼은 카드 하단 전체 너비로 둔다.

---

## 9. 접근성 기준

- 초록색 점만으로 상태를 전달하지 않고 텍스트를 함께 표시한다.
- 버튼과 입력 필드는 키보드 포커스 상태를 표시한다.
- 주요 텍스트와 배경의 대비를 충분히 확보한다.
- 에러 메시지는 색상과 문장으로 함께 안내한다.
