# 03. 개발 환경 구성

## 필요 도구

| 도구 | 용도 | 확인 명령 |
|------|------|-----------|
| Node.js 20+ | 프론트엔드 빌드 | `node -v` |
| Python 3.11+ | 백엔드 실행 | `python --version` |
| Git | 버전 관리 | `git --version` |
| GitHub CLI | 저장소/PR 관리 | `gh --version` |

---

## 1. 저장소 설정

GitHub에서 `namu` 저장소를 생성한 뒤 연결한다.

```bash
gh auth login
git init
git branch -M main
git remote add origin https://github.com/jfktwo/namu.git
git push -u origin main
```

---

## 2. 프론트엔드 초기화

```bash
npm create vite@latest frontend -- --template react-ts
cd frontend
npm install
npm run dev
```

로컬 주소: `http://localhost:5173`

---

## 3. 백엔드 초기화

```bash
cd backend
python -m venv .venv

# Windows
.venv\Scripts\activate

pip install fastapi uvicorn python-dotenv
uvicorn app.main:app --reload
```

로컬 주소: `http://localhost:8000`  
API 문서: `http://localhost:8000/docs`

---

## 4. Supabase 설정

1. [supabase.com](https://supabase.com) 에서 프로젝트 생성
2. `Settings > API` 에서 아래 값 확인

| 항목 | 위치 |
|------|------|
| Project URL | Settings > API > Project URL |
| anon key | Settings > API > Project API keys |
| DB 연결 문자열 | Settings > Database > Connection string |

---

## 5. 환경 변수

프로젝트 루트에 `.env` 파일을 만들고, `.gitignore`에 등록한다.

```env
# Supabase
SUPABASE_URL=https://xxxxxx.supabase.co
SUPABASE_ANON_KEY=xxxxxx

# Backend
DATABASE_URL=postgresql://postgres:password@db.xxxxxx.supabase.co:5432/postgres

# Kakao Map
KAKAO_MAP_API_KEY=xxxxxx
```

프론트엔드는 Vite 환경변수 규칙을 따른다 (`VITE_` 접두어).

```env
# frontend/.env
VITE_API_BASE_URL=http://localhost:8000
VITE_KAKAO_MAP_API_KEY=xxxxxx
```

---

## 6. 폴더 생성

```bash
mkdir frontend backend database
mkdir database\migrations database\seed
```

실제 폴더 구조는 [02.소프트웨어 구조.md](02.소프트웨어%20구조.md)를 따른다.
